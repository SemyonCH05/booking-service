# booking-service

Генерация слотов выполняется автоматически и продлевается каждый день, чтобы всегда было доступна недельное окно дат, так как генерация слотов во время запроса будет увеличивать время ответа сервера. Свободные слоты хранятся в БД заранее и не генерируются на каждом запросе GET /rooms/{roomId}/slots/list. При старте и далее раз в 24 часа выполняется фоновая подгрузка слотов: для каждого расписания берётся дата последнего сгенерированного слота или следующего дня, если слотов ещё нет и дозаполняется диапазон в UTC до today + 7 дней по дням недели и временным границам расписания. Генерация выполняется через INSERT ... ON CONFLICT DO NOTHING, поэтому процесс идемпотентный и не создаёт дублей. Благодаря этому путь сводится к чтению готовых слотов и выдаче ответа с минимальной задержкой.
